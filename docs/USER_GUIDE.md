# NytroAI User Guide

Welcome to NytroAI! This guide will help you validate your training assessments quickly and easily.

---

## Getting Started

### Step 1: Upload Your Assessment

1. Click **"Upload Assessment"** in the dashboard
2. Drag and drop your PDF file (or click to browse)
3. Enter the **Unit Code** (e.g., BSBWHS332X)
4. Click **"Start Validation"**

**Tips:**
- Make sure your file is in PDF format
- File size should be under 50MB
- Use the correct unit code from training.gov.au

---

### Step 2: Wait for Processing

Your assessment goes through two stages:

**1. Document Processing** (30-60 seconds)
- AI reads and indexes your assessment
- Progress bar shows status

**2. Validation** (1-2 minutes)
- AI checks against all unit requirements
- You'll see "Processing in Background"

**What to do while waiting:**
- Keep the browser tab open
- Don't refresh the page
- You can upload more assessments

---

---

### Step 4: Use Smart Questions

For each gap (Partial or Not Met), NytroAI generates a **Smart Question** you can add to your assessment.

#### How to Use Smart Questions

1. **Review the question** - Make sure it fits your assessment style
2. **Copy the question** - Click the copy button
3. **Add to your assessment** - Paste into your document
4. **Customize if needed** - Adjust wording to match your context

#### Regenerate Questions

Don't like the question? Click **"Regenerate"** to get a new one.

**Add context** to get better questions:
- "Focus on workplace safety"
- "Make it scenario-based"
- "Suitable for beginners"

## Dashboard Features

### Overview Cards

**Total Validations** - How many assessments you've validated

**Completed** - Validations that finished successfully

**In Progress** - Currently processing

**Failed** - Had errors (click to see why)

### Validation List

Each validation shows:
- **Unit Code** - Which unit was validated
- **Status** - Current processing stage
- **Progress** - Percentage complete
- **Date** - When it was uploaded

**Actions:**
- **View Results** - Double-click to see details
- **Delete** - Remove validation (can't be undone!)
- **Refresh** - Update status

---

## Understanding Results

### Validation Summary

At the top of results, you'll see:
- **Total Requirements** - How many things to check
- **Met** - How many are fully addressed
- **Partial** - How many need improvement
- **Not Met** - How many are missing

### Requirement Details

For each requirement, you'll see:

**1. Requirement Text**
- The exact wording from the unit

**2. Status**
- Met, Partial, or Not Met

**3. Reasoning**
- Why the AI made this decision
- Which questions in your assessment address this
- What's missing or could be improved

**4. Smart Question**
- A question you can add to fill the gap
- Only shown for Partial and Not Met

## Exporting Results

### Export Options

**PDF Report** - Professional compliance report
- Summary page
- Detailed findings
- Recommendations
- Ready to print or share

**Excel Spreadsheet** - Data for analysis
- All requirements in rows
- Status, reasoning, questions in columns
- Easy to filter and sort

**JSON Data** - For developers
- Complete validation data
- Machine-readable format

### How to Export

1. Open validation results
2. Click **"Export"** button
3. Choose your format
4. File downloads automatically

## Tips for Better Results

### Before Uploading

‚úÖ **Use clear, well-structured PDFs**
- Avoid scanned images
- Use actual text (not images of text)
- Organize questions clearly

‚úÖ **Include all assessment components**
- Knowledge questions

‚úÖ **Use the correct unit code**

### After Validation

‚úÖ **Review AI reasoning carefully**
- AI is a tool, not a replacement for judgment
- Check if the AI understood your questions correctly
- Look for false positives/negatives

‚úÖ **Customize smart questions**
- Adjust wording to match your style
- Add context specific to your industry
- Make sure they fit your assessment format

‚úÖ **Validate again after changes**
- Upload the updated assessment
- Compare results to see improvement
- Track your progress over time

## Common Workflows

### Workflow 1: New Assessment

1. Upload draft assessment
2. Review validation results
3. Add smart questions for gaps
4. Update assessment document
5. Upload again to verify
6. Export final report

### Workflow 2: Existing Assessment

1. Upload current assessment
2. Identify gaps and weaknesses
3. Improve existing questions
4. Add new questions for gaps
5. Validate updated version
6. Compare before/after results

### Workflow 3: Multiple Units

1. Upload assessment for Unit 1
2. While processing, upload Unit 2
3. Review results as they complete
4. Track all validations in dashboard
5. Export reports for each unit
6. Compile into compliance package
