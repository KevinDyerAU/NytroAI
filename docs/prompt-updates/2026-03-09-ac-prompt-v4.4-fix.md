# Assessment Conditions Prompt v4.4 - Fix for AC4/AC5 False "Not Met"

**Date**: 2026-03-09
**Prompt ID**: 256 (replaces 254)
**Version**: v4.4 (was v4.3)
**Author**: Manus Agent (requested by Adam Murphy)

## Issue

Assessment conditions AC4 (language, literacy and numeracy requirements) and AC5 (resources for assessment) were being incorrectly marked as "Not Met" with the reasoning:

> "The specific assessment condition to be validated was not provided in the prompt, therefore validation cannot be performed."

## Root Cause

The v4.3 default prompt (id=254) did not include `{{requirement_number}}` or `{{requirement_text}}` template placeholders. The `trigger-validation-unified` function calls `replacePromptVariables()` which substitutes these placeholders with the actual requirement data before sending to the LLM:

```typescript
promptText
  .replace(/{{requirement_number}}/g, requirement.number || '')
  .replace(/{{requirement_text}}/g, requirement.text || '')
  .replace(/{{requirement_type}}/g, requirement.type || '')
  .replace(/{{unit_code}}/g, unitCode || '')
```

Without these placeholders in the prompt text, the LLM received the general validation instructions but never received the specific assessment condition it was supposed to validate. When the LLM couldn't infer the condition from the document context alone, it returned "not provided in the prompt."

## Fix Applied

The v4.4 prompt adds an explicit section at the top of the prompt text:

```
ASSESSMENT CONDITION BEING VALIDATED:
Condition Number: {{requirement_number}}
Condition Text: {{requirement_text}}
```

And reinforces this throughout the prompt:
- "You are validating the SPECIFIC assessment condition shown above ({{requirement_number}}) against the assessment documents."
- "Read the specific assessment condition {{requirement_number}} shown above."

The system instruction was also updated to explicitly state:
- "You will receive the condition number and condition text in the prompt. Your job is to validate THAT SPECIFIC condition."
- "You are validating ONE specific assessment condition at a time."

## Database Changes

| Action | Table | ID | Details |
|--------|-------|-----|---------|
| UPDATE | prompts | 254 | Set `is_default = false` |
| INSERT | prompts | 256 | New v4.4 prompt with `is_default = true` |

## Compatibility

The prompt is designed to work with both GPT models and Google Gemini models. The `{{placeholder}}` syntax is compatible with the `replacePromptVariables()` function in `trigger-validation-unified/index.ts`.

## Future Improvement

The `requirements-fetcher.ts` currently returns the entire AC text from `UnitOfCompetency.ac` as a single requirement (number: "1"). Individual AC requirements (AC 1, AC 2, AC 3, etc.) exist in the `assessment_conditions_requirements` table but are not used by the `trigger-validation-unified` flow. A future improvement would be to update the fetcher to use individual AC rows for more granular validation.
